IFTAR Acerca GitHub

¿Qué es IFTAR?

IFTAR (Image Forge To Art Retro) es una API web que transforma imágenes comunes (JPG, PNG.) en arte de terminal. Nace como un fork del proyecto radare/tiv (Terminal Image Viewer), adaptándolo a una arquitectura web con 5 modos de renderizado distintos.

El proyecto está desarrollado en Node.js con Express, usando sharp para el procesamiento de imágenes y node-canvas para el renderizado final. Cada modo implementa un algoritmo específico de reducción de color y mapeo visual.

Pipeline de procesamiento

Todos los modos comparten el mismo flujo de procesamiento. La diferencia está en cómo se procesa cada píxel dentro del bucle de renderizado:

  1. Carga y rotación — La imagen se carga con sharp y se rota automáticamente según metadatos EXIF.
  2. Ajuste de contraste — Si está activado el autocontraste, se aplica ecualización de histograma.
  3. Redimensionado — Se escala la imagen al ancho solicitado (20 a 500 columnas) manteniendo la proporción original.
  4. Extracción de píxeles — Se elimina el canal alfa y se obtiene un buffer RAW con los valores RGB de cada píxel.
  5. Renderizado — Se crea un canvas y se itera píxel por píxel (de 2 en 2 en altura) aplicando la lógica del modo seleccionado.
  6. Streaming — El canvas se convierte en un stream PNG o JPG y se envía como respuesta HTTP.

Modos de renderizado

Modo ASCII

Convierte cada píxel a un carácter que representa su nivel de brillo. Los píxeles más oscuros se representan con caracteres "llenos" y los más claros con caracteres "vacíos".

Lógica

Para cada píxel se calcula la luminancia usando el estándar BT.709 y se mapea a un índice dentro de la paleta de caracteres seleccionada.

Paletas disponibles: original (10 chars), conservative (13 chars), expanded (16 chars), detailed (12 chars).

Fórmula clave

Ver fórmula de luminancia BT.709

L = 0.2126 × R + 0.7152 × G + 0.0722 × B

Donde R, G y B son los valores del píxel (0-255).

El carácter se selecciona con: índice = min(⌊L / (255 / N)⌋, N - 1), donde N es el tamaño de la paleta.

Ver métodos alternativos de luminancia

NTSC: L = 0.299 × R + 0.587 × G + 0.114 × B

Gamma: L = (0.299 × R2.2 + 0.587 × G2.2 + 0.114 × B2.2)1/2.2

Modo ANSI

Reduce la imagen a los 8 colores básicos de terminal. Cada celda (de 2 píxeles de alto) usa el píxel inferior como color de fondo y el superior como color de texto, simulando el efecto de un terminal clásico.

Lógica

Se calcula la distancia entre el color de cada píxel y los 8 colores de la paleta ANSI usando distintos métodos cromáticos. El color más cercano es el que se asigna. El color de primer plano se oscurece restando 20 a cada canal RGB para diferenciarlo del fondo.

Fórmulas clave

Ver métodos de distancia de color

Clásico (euclidiana): d = √((R₁−R₂)² + (G₁−G₂)² + (B₁−B₂)²)

Perceptual (ponderado): d = √(2×(R₁−R₂)² + 4×(G₁−G₂)² + 3×(B₁−B₂)²)

CIEDE2000: Distancia en espacio de color Lab con correcciones perceptuales avanzadas.

CMC l:c: Distancia en espacio Lab con pesos para luminancia y cromaticidad.

Ver algoritmo de dithering Floyd-Steinberg

El error de cuantificación se distribuye a los píxeles vecinos:

  • (x+1, y) → 7/16 del error
  • (x−1, y+1) → 3/16 del error
  • (x, y+1) → 5/16 del error
  • (x+1, y+1) → 1/16 del error

Modo Grey

Convierte la imagen a 24 tonos de gris usando la paleta xterm (índices 232 a 255). Es el modo ideal para lograr un efecto monocromático limpio y detallado.

Lógica

Se calcula la luminancia de cada píxel con BT.709 y se mapea al tono de gris más cercano dentro de la paleta de 24 niveles. Soporta dithering para suavizar la transición entre tonos.

Fórmula clave

Ver fórmula de mapeo a grises

L = BT.709(R, G, B)

índice = max(0, min(231, round(L × 232 / 256)))

El color final es GREY_PALETTE[índice + 232]

Modo 256

Utiliza la paleta completa de 256 colores xterm para una reproducción cromática más fiel. Es el modo con mayor riqueza de color después del RGB.

Lógica

Cada píxel se mapea al color más cercano de la paleta de 256 colores usando métodos de distancia cromática. Para optimizar el rendimiento, implementa un caché LRU (Least Recently Used) de 8000 entradas que evita recalcular el mapeo de colores repetidos.

Detalles técnicos

Ver funcionamiento del caché LRU

Cuando un color RGB ya fue mapeado previamente, se devuelve el resultado desde el caché en lugar de iterar los 256 colores de la paleta. Si el caché supera las 8000 entradas, se elimina el elemento menos usado.

Modo RGB

Actúa como un passthrough de color: los píxeles conservan su color original sin reducción de paleta. Solo se aplican ajustes de brillo y dithering opcional.

Lógica

Los valores RGB se mantienen intactos con un simple clamping al rango 0-255. Este modo es útil para cambiar el formato de salida (PNG a JPG) o aplicar efectos suaves sin perder fidelidad cromática.

Fórmula clave

Ver fórmula de clamping

R' = clamp(R + brightness, 0, 255)

G' = clamp(G + brightness, 0, 255)

B' = clamp(B + brightness, 0, 255)
